\chapter{PROJECT DESIGN AND IMPLEMENTATION}
\label{ch:Project Design and Implementation}
This chapter presents the design and implementation of the autonomous parcel delivery navigation system. It covers the development approach, performance metrics, software architecture, algorithm design, simulation setup, and prototype demonstration. The chapter explains the step-by-step development process and testing methods used to ensure the system operates reliably and efficiently in complex indoor environments.
\section{Proposed Design Methodology}

The proposed design methodology develops an autonomous parcel delivery system by integrating a web-based management platform with the ROS2 navigation framework, simulation environment, visualization tools, and autonomous navigation algorithms. The system enables users to assign delivery tasks, monitor the robot in real time, manage maps, and visualize robot operations through a unified interface.

Figure~\ref{overall} illustrates the overall system architecture. The top layer consists of a web interface that provides live robot telemetry, task assignment, map management, and delivery log monitoring. The web interface communicates with a Flask-based backend server, which manages the application logic, REST APIs, database operations, and acts as a bridge between the web platform and ROS2. The ROS2 communication layer coordinates the interaction among the navigation stack, Gazebo simulation, RViz visualization, and the TurtleBot3 robot model. The navigation stack consists of the Improved A* path planning algorithm, Particle Swarm Optimization (PSO)-based path smoothing, and a Model Predictive Controller (MPC) for trajectory tracking. Gazebo provides the simulation environment for testing robot navigation, while RViz visualizes the generated maps, planned paths, and live robot telemetry.

\subsection*{Phase 1: Navigation Algorithm Development}

The Improved A* path planning algorithm was implemented to generate collision-free global paths within the simulated environment. The generated paths were further enhanced using Particle Swarm Optimization (PSO)-based path smoothing to obtain shorter, smoother, and more feasible trajectories for autonomous navigation.

\subsection*{Phase 2: Web-Based Management Platform Development}

A Flask-based web application was developed to provide an intuitive interface for users to assign delivery tasks, monitor live robot telemetry, manage occupancy maps, and access delivery logs. The interface serves as the primary interaction point between the user and the autonomous delivery system.

\subsection*{Phase 3: Backend and ROS2 Integration}

The backend server was integrated with ROS2 to establish seamless communication between the web platform and the robotic system. User commands, delivery goals, and map management requests are forwarded to ROS2, while robot status and telemetry information are continuously transmitted back to the web interface.

\subsection*{Phase 4: Model Predictive Control Implementation}

A Model Predictive Controller (MPC) was implemented to accurately track the optimized navigation path generated by the planning module. The controller computes appropriate velocity commands while satisfying the robot's kinematic constraints, resulting in smooth and stable autonomous motion.

\subsection*{Phase 5: Simulation and Visualization Integration}

The complete navigation stack was integrated with Gazebo and RViz. Gazebo provides a realistic simulation environment for evaluating robot performance, whereas RViz offers real-time visualization of occupancy maps, robot pose, planned trajectories, and navigation status during autonomous deliveries.

\subsection*{Phase 6: System Deployment and Validation}

The complete autonomous parcel delivery system was deployed using Ubuntu 22.04, ROS2, Gazebo, and the TurtleBot3 robot model equipped with LiDAR and RGB camera sensors. End-to-end testing was performed to validate task execution, autonomous navigation, path tracking, and communication between the web interface and the robotic platform.

\section{Analysis Procedure}


The analysis process is designed to analyze the performance, reliability and efficiency of the autonomous navigation system in various operation conditions. Quantitative metrics are applied to the performance evaluation in which the quality of path planning, the effectiveness of optimization, the accuracy of the trajectory tracking, and the efficiency of the computations are evaluated.

\subsection{Path Planning Performance Metrics}

The quality and feasibility of generated paths are examined in determining performance of path planning. This test is aimed at collision-free navigation verification and reduction of path length, as well as its computational efficiency.
Path planning performance is measured using the following metrics:

\subsubsection{Path Length}


Path length is a distance that a robot traverses on the plan. It is computed by adding the breakeven distance among successive waypoints. The shortest routes are desirable because they save on time of travel and energies.
\subsubsection{Collision-Free Validation}

The consequent pathway is confirmed with the boundaries of obstacles by carrying out geometric intersection detection algorithms. The path must ensure that there is a small safety margin compared to all the hazards in an attempt to prevent the occurrence of collisions when running.

\subsubsection{Waypoint Density}

The density of the waypoints measures the number of intermediate nodes that are put in place along the route. Excessive number of waypoints means the system is prone to high computing loads in tracking the trajectories and insufficient number of waypoints may undermine the accuracy of the route especially in complex environments.
\subsection{Path Optimization and Smoothness Evaluation}

The path optimization analysis discussed below focuses on path optimization done after the global path generation. This will be done to achieve a better smoothness and minimise sudden direction changes hence, to better track trajectory performance and mechanical stress of the robot.  

The efficacy of the optimization is assessed using the following measures:  

\subsubsection{Curvature Continuity}

The change in curvature in successive path segments is studied to determine the directional consistency. Reduction in the curvature discontinuities is an indication of a smoother transition, which makes the trajectory applicable to the differential-drive robots.  

\subsubsection{Turning Angle Analysis}

Angles between path segments are calculated between successive path segments. Smaller turning angles represent smoother routes and it follows that such routes demand less aggressive steering inputs.  

\subsubsection{Waypoint Reduction Ratio}


Path pruning gets rid of unnecessary waypoints created by global planner. The percentage of waypoints that have been removed over the number of original waypoints is considered the pruning efficiency, and the higher ratios the more effectively waypoints have been simplified without affecting the quality of the paths.

\subsection{Trajectory Tracking Performance Metrics}
Trajectory tracking performance measurement is necessary to determine how the robot follows the optimal path calculated using Model Predictive Control(MPC).


\subsubsection{Tracking Error}
The difference between the required trajectory and the real route of the robot is referred to as the tracking error. This measure is determined as the perpendicular distance between the current location of the robot and the closest location at the reference trajectory. A decrease in the error of tracking is a factor of the superior controller effectiveness.

\subsubsection{Orientation Error}

Orientation error measures the difference between the heading of the robot and the heading that is needed according to the trail that it is on. Correct orientation is essential with the differential-drive platforms because any error in heading can be disastrous to following the path correctly.

\subsubsection{Velocity Tracking Accuracy}

Velocity tracking is used to measure how fast the robot is following the velocity profile as specified by the MPC controller. The constant speed monitoring of the velocity is not only a requirement of smooth movement, but it also assures the predictability of the dynamic behaviour of the robotic system.
\subsection{Computational Efficiency Analysis}

Computational efficiency is evaluated to ensure real-time navigation feasibility. This analysis examines the processing time required for various system components:

\begin{itemize}[noitemsep]
	\item Path planning execution time
	\item PSO optimization convergence time

	\item Visualization, server processing delay
\end{itemize}

Computational efficiency is considered in order to determine the feasibility of real-time navigation. This discussion examines the time required to process the different parts of the system:

\begin{itemize}[noitemsep]

\section{Implementation Procedure}

This section describes the step-by-step implementation process followed to develop the autonomous parcel delivery navigation system. The implementation follows the six-phase methodology described in section 3.2

\subsection{Phase 1: Path Planning Algorithm Implementation}

\subsubsection{A* Algorithm Implementation}

The A* path planning algorithm is implemented in Python using a grid-based representation of the workspace. The environment is discretized into a uniform grid with configurable resolution. Each grid cell is classified as free space or occupied based on obstacle positions. The algorithm maintains open and closed sets for node exploration and uses a priority queue to select nodes with minimum cost function:

\begin{equation}
	f(n) = g(n) + h(n)
\end{equation}

where $g(n)$ is the actual cost from the start node to the current node, and $h(n)$ is the heuristic estimate from the current node to the goal node.
\subsubsection{Path Pruning Implementation}

Path pruning is applied to reduce redundant waypoints generated by the A* algorithm. The pruning algorithm iterates through consecutive waypoints and removes intermediate points that do not contribute to path quality. A line-of-sight check is performed between non-consecutive waypoints to verify collision-free connectivity.

\subsubsection{PSO-Based Path Smoothing Implementation}

The smoothness of navigational trajectories has been improved with the help of Particle Swarm Optimization that has been utilized. Every single, or particle, represents a possible flattened path, where the nodes are the constituent positional variables. The swarm then, following a series of repetitions, recalculates the positions of the particles through velocity equations and simultaneously measures cost functions which combine measures of path length, curvature and obstacle proximity.

\section{Details of Simulations / Mathematical Modeling}

This section presents the mathematical formulations and simulation models used to develop and evaluate the autonomous navigation system. The modeling covers robot kinematics, path planning algorithms, optimization techniques, and control formulations.

\subsection{Robot Kinematic Modeling}

The TurtleBot3 Burger robot used in this project follows a differential drive kinematic model. The robot's motion is described by the following equations:

\begin{align}
	\dot{x} &= v \cos(\theta) \label{eq:kinematic_x} \\
	\dot{y} &= v \sin(\theta) \label{eq:kinematic_y} \\
	\dot{\theta} &= \omega \label{eq:kinematic_theta}
\end{align}

where:
\begin{itemize}[noitemsep]
	\item $x, y$ are the robot's position coordinates
	\item $\theta$ is the robot's heading angle
	\item $v$ is the linear velocity
	\item $\omega$ is the angular velocity
\end{itemize}

These equations are used in both the path planning validation and trajectory tracking using MPC. The discrete-time formulation is obtained using Euler integration for numerical simulation.

\subsection{Mathematical Modeling of Path Planning}

The path planning process is modeled as a graph search problem where the environment is divided into a discrete grid. The A* algorithm evaluates nodes using the cost function:

\begin{equation}
	f(n) = g(n) + h(n)
	\label{eq:astar_cost}
\end{equation}

Where:
\begin{itemize}[noitemsep]
	\item $g(n)$ = actual cost from start node to current node
	\item $h(n)$ = heuristic estimate from current node to goal node
\end{itemize}

The Euclidean distance heuristic is used:

\begin{equation}
	h(n) = \sqrt{(x_{goal} - x_{current})^2 + (y_{goal} - y_{current})^2}
	\label{eq:euclidean_heuristic}
\end{equation}

Dynamic programming techniques are used to store optimal sub-path solutions, improving computational efficiency through memoization.

\subsection{Optimization Formulation of PSO Smoothing}

Particle Swarm Optimization (PSO) is applied to smooth the initially generated path and remove unnecessary sharp turns. Each particle represents a candidate smoothed path, and the swarm optimizes a multi-objective cost function:

\begin{equation}
	J = w_1 \cdot L + w_2 \cdot C + w_3 \cdot P
	\label{eq:pso_cost}
\end{equation}

Where:
\begin{itemize}[noitemsep]
	\item $L$ = total path length
	\item $C$ = curvature or smoothness factor
	\item $P$ = obstacle penalty function
	\item $w_1, w_2, w_3$ = weighting coefficients
\end{itemize}

Particle position and velocity updates follow standard PSO equations:

\begin{align}
	v_i(t+1) &= w \cdot v_i(t) + c_1 \cdot r_1 \cdot (p_{best,i} - x_i(t)) + c_2 \cdot r_2 \cdot (g_{best} - x_i(t)) \label{eq:pso_velocity} \\
	x_i(t+1) &= x_i(t) + v_i(t+1) \label{eq:pso_position}
\end{align}

where:
\begin{itemize}[noitemsep]
	\item $w$ = inertia weight
	\item $c_1, c_2$ = cognitive and social coefficients
	\item $r_1, r_2$ = random values in [0,1]
	\item $p_{best,i}$ = personal best position
	\item $g_{best}$ = global best position
\end{itemize}

This smoothing step ensures the path is suitable for real robot motion and improves trajectory tracking performance.

\subsection{Mathematical Formulation of MPC Controller}

Model Predictive Control (MPC) is implemented for trajectory tracking. MPC predicts future robot motion over a prediction horizon and computes optimal control inputs. The optimization problem is formulated as:

\begin{equation}
	\min_{u} \sum_{k=0}^{N-1} \left( \|x_k - x_{ref}\|^2_Q + \|u_k\|^2_R \right)
	\label{eq:mpc_cost}
\end{equation}

subject to:
\begin{itemize}[noitemsep]
	\item Robot kinematic constraints (Equations \ref{eq:kinematic_x}--\ref{eq:kinematic_theta})
	\item Velocity limits: $v_{min} \leq v \leq v_{max}$, $\omega_{min} \leq \omega \leq \omega_{max}$
	\item Obstacle avoidance constraints
\end{itemize}

where:
\begin{itemize}[noitemsep]
	\item $x_k$ = predicted robot state at time step $k$
	\item $x_{ref}$ = reference trajectory
	\item $u_k$ = control input $(v, \omega)$
	\item $Q, R$ = weighting matrices for state error and control effort
	\item $N$ = prediction horizon
\end{itemize}
The MPC controller continuously updates control commands, ensuring accurate trajectory tracking and smooth robot motion.

\subsection{Algorithm Parameters and Configuration}

\subsubsection*{A* Algorithm Configuration}

The A* path planning algorithm is configured with the following adaptive parameters:
\begin{description}[leftmargin=0cm, labelindent=0cm, itemsep=0.3em, font=\normalfont]
	\item[Grid Resolution:] The grid resolution adapts to workspace dimensions to balance computational efficiency and path accuracy:
	\begin{equation}
		\text{resolution} = \min(\text{width}, \text{height}) \times 0.01
	\end{equation}
	
	\item[Safety Margin:] A dynamic safety margin is applied around obstacles to prevent collisions:
	\begin{equation}
		\text{safety\_margin} = \text{resolution} \times 0.2 \text{ meters}
	\end{equation}
	
	\item[Motion Model:] The robot uses an 8-directional motion model with diagonal movement cost of $\sqrt{2} \times \text{resolution}$.
	
	\item[Grid Indexing Formula:] Each cell is assigned a unique index for efficient lookup:
	\begin{equation}
		\text{index} = y_{\text{index}} \times x_{\text{width}} + x_{\text{index}}
	\end{equation}
	
	\item[Heuristic Function:] Euclidean distance is used as the admissible heuristic for optimal pathfinding.
\end{description}

\subsubsection*{Dynamic Programming Configuration}

The cell decomposition-based Dynamic Programming planner uses the following key components:

\begin{description}[leftmargin=0cm, labelindent=0cm, itemsep=0.3em, font=\normalfont]
	\item[Memory Table:] A 2D memoization table of size $n_{\text{cells}} \times n_{\text{cells}}$ stores precomputed paths between cell pairs for efficient retrieval.
	
	\item[Map Hashing:] SHA-256 hashing detects environment changes. When the map hash matches the saved hash, stored paths are reused; otherwise, replanning occurs.
	
	\item[Cell Center Calculation:] Cell centroids are computed as:
	\begin{align}
		x_{\text{center}} &= \frac{\min_x + \max_x}{2} \\
		y_{\text{center}} &= \frac{\min_y + \max_y}{2}
	\end{align}

\subsection{Algorithm Selection and Justification}

\begin{description}[leftmargin=0cm, labelindent=0cm, itemsep=0.5em]
	\item[A* Algorithm:] Selected for global path planning due to its optimality guarantees and heuristic-guided efficiency. The algorithm ensures shortest path generation while minimizing computational overhead.
	
	\item[Particle Swarm Optimization:] Chosen for path smoothing because of its global optimization capability and ability to handle multi-objective cost functions balancing path length, smoothness, and obstacle clearance.
	
	\item[Model Predictive Control:] Selected for trajectory tracking due to its predictive capability, which considers future states and constraints. MPC enables smooth control command generation and accurate path following under kinematic constraints.
	
	\item[Dynamic Programming with Cell Decomposition:] Implemented to provide systematic grid-based exploration with memoization benefits. This approach excels in structured environments and enables rapid replanning through stored path segments.
\end{description}
